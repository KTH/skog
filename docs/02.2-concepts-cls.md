# Continuation-style

Continuation-style is a way of passing variables to functions without actually passing them. Since this technique can confuse people (pass variables without passing them), it must be used in a very controlled way and in few scenarios. Logging is one of those valid use cases.

Let's see how more or less this works in Skog.

First, you declare a **global** `log` variable.

```js
const log = require('skog')
```

However, this variable might have different values. Which means that the variable has different behavior depending on the **run-time call-stack of where it is accessed**

Look at the following code:

```js
const log = require('skog')

function a () {
  log.info('Hello from A')
}

function b () {
  log.info('Hello from B')
  a()
}

log.info('Hello from outside')
a()
b()
```

This code will log:

```
Hello from outside
Hello from A
Hello from B
Hello from A
```

Let's write the **call-stack** of every log line, so the function that called the current function. This is something Skog tracks internally.

```
Hello from outside     (call-stack: global)
Hello from A           (call-stack: global -> a)
Hello from B           (call-stack: global -> b)
Hello from A           (call-stack: global -> b -> a)
```

As you can see the first `Hello from A` and the second one have different call-stacks.

When you call `skog.child` you are telling Skog to use a different logger from that point.

```js
skog.child({ req_id: 1 }, () => {
  // Here `log` have a different value
  log.info('Hello')
  // Same with the value of `log` when we call another function:
  b()
})
```
